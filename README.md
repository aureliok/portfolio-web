# Portfolio Web

Personal portfolio website built with Astro, React and Tailwind CSS. Features a bilingual interface (English and Portuguese), project showcase with tag filtering, and an AI-powered chatbot (coming soon).

Live at: [aureliok-dev.vercel.app](https://aureliok-dev.vercel.app)

---

## Tech Stack

- **[Astro](https://astro.build/)** — static site generator optimized for content-focused websites
- **[React](https://react.dev/)** — used for interactive components (project filter, chatbot)
- **[Tailwind CSS v4](https://tailwindcss.com/)** — utility-first CSS framework
- **[TypeScript](https://www.typescriptlang.org/)** — type safety across the project

---

## Architecture

The site is statically generated by Astro and deployed on Vercel. Dynamic data (projects) is fetched at runtime from the [Portfolio API](https://github.com/aureliok/portfolio-api).

```
portfolio-web/
  src/
    components/       ← Reusable React and Astro components
    i18n/             ← Translation files and utility functions
      en.ts           ← English translations
      pt.ts           ← Portuguese translations
      utils.ts        ← Language detection and translation helpers
    layouts/          ← Page layouts (nav, footer)
    pages/            ← Astro pages, organized by locale
      en/             ← English pages
      pt/             ← Portuguese pages
      index.astro     ← Root redirect with language detection
    styles/           ← Global CSS
  public/             ← Static assets (resume PDFs, favicon)
```

### i18n

The site uses Astro's built-in i18n routing with `prefixDefaultLocale: true`, meaning all pages are prefixed with their locale:

```
/en          → English home
/pt          → Portuguese home
/en/about    → English about
/pt/about    → Portuguese about
```

Language detection is done client-side on the root `/` page using `navigator.language`, redirecting the user to their preferred language automatically.

### Interactive components

Static Astro pages handle most of the site, but React components are used where interactivity is needed:

- **`ProjectsGrid.tsx`** — fetches projects from the API at runtime, supports tag filtering
- **`MobileMenu.tsx`** — hamburger menu for mobile navigation

## Running Locally

### Prerequisites

- Node.js 18+
- The [Portfolio API](https://github.com/yourusername/portfolio-api) running locally

### Setup

1. Clone the repository:
```bash
git clone https://github.com/yourusername/portfolio-web.git
cd portfolio-web
```

2. Install dependencies:
```bash
npm install
```

3. Create a `.env` file based on `.env.example`:
```bash
cp .env.example .env
```

4. Fill in the values:
```
PUBLIC_API_URL=http://localhost:8000
```

5. Start the dev server:
```bash
npm run dev
```

6. Open `http://localhost:4321`

## Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `PUBLIC_API_URL` | Base URL of the Portfolio API | `http://localhost:8000` |

Note: The `PUBLIC_` prefix is required by Astro for environment variables accessible in the browser.

## Deployment

The site is deployed on [Vercel](https://vercel.com) with automatic deployments on every push to `main`.

### Steps

1. Import the repository in Vercel
2. Vercel will automatically detect it as an Astro project
3. Add the environment variable in Vercel's dashboard:
   - `PUBLIC_API_URL` → your production API URL (e.g. `https://portfolio-api.onrender.com`)
4. Click Deploy

### Updating content

- **Projects** — managed via the Portfolio API admin endpoints
- **Bio, skills, experience, education** — edit the page files directly in `src/pages/en/` and `src/pages/pt/`
- **Translations** — edit `src/i18n/en.ts` and `src/i18n/pt.ts`
- **Resume** — replace `public/resume-en.pdf` and `public/resume-pt.pdf`
